# 任务拆解技能

将复杂任务拆解为可执行、可追踪的子任务序列。

## 工作流程

1. **理解任务** - 分析用户需求，识别核心目标
2. **识别步骤** - 按逻辑顺序拆解为子任务
3. **分析依赖** - 确定步骤间的依赖关系
4. **分配角色** - 匹配合适的技能/Agent
5. **输出规划** - 生成结构化任务清单

## 拆解原则

| 原则 | 说明 |
|------|------|
| **MECE** | 相互独立，完全穷尽 |
| **最小单元** | 每个子任务有明确输入输出 |
| **依赖明确** | 清晰标注步骤依赖 |
| **角色匹配** | 分配给合适的技能 |

## 输入参数

| 参数 | 必填 | 说明 |
|------|------|------|
| `task` | 是 | 任务描述 |
| `context` | 否 | 背景信息 |
| `constraints` | 否 | 限制条件 |
| `preferredSkills` | 否 | 优先使用的技能 |
