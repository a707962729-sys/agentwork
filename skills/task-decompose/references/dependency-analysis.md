# 依赖分析技巧

## 依赖类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **强依赖** | B 必须等 A 完成 | 测试依赖开发完成 |
| **弱依赖** | B 可以并行，但 A 提供输入 | UI 设计和后端开发 |
| **无依赖** | 完全独立 | 文档编写和代码开发 |

---

## 依赖可视化

### DAG（有向无环图）
```
    [需求分析]
        ↓
    [系统设计] ←─ [UI设计]
        ↓           ↓
    [后端开发]   [前端开发]
        ↓           ↓
    [接口联调] ────→
        ↓
    [测试验收]
        ↓
    [上线部署]
```

### 关键路径
从起点到终点的最长路径，决定最短完成时间。

---

## 并行化策略

### 可并行的场景
- 前端开发 ‖ 后端开发
- UI 设计 ‖ 数据库设计
- 文档编写 ‖ 代码开发

### 必须串行的场景
- 需求 → 设计
- 开发 → 测试
- 测试 → 上线

---

## 依赖管理最佳实践

1. **尽早识别依赖** - 拆解时标记
2. **减少强依赖** - 通过解耦降低耦合
3. **并行最大化** - 能并行的都并行
4. **关键路径优先** - 资源倾斜给关键路径
5. **预留缓冲** - 关键依赖点预留时间
